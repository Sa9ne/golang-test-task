# Инструкция по запуску микросервиса

Для проверки данного микросервиса необходимо добавить env файл:
```
DATABASE_URL=postgres://ПользовательБД(По умолчанию: postgres):ПарольОтБд@db:5432/НазваниеБД

POSTGRES_PASSWORD=ПарольОтБд

POSTGRES_USER=ПользовательБД(По умолчанию: postgres)

POSTGRES_DB=НазваниеБД
``` 

Далее из папки golang-test-task прописать команду для запуска Docker:
```
docker compose up --build
```

После того как докер запустился необходимо проверить работу контейнера golang, и если он не работает то перейти в него и запустить (У меня есть такая проблема и я не уверен, только на моем устройстве ли это). После нажатия на кнопку микросервис включиться без проблем

Написать микросервис на Go с REST эндпоинтом, принимающем на вход одно число
Число должно сохраняться в БД
В ответ должен возвращаться список отсортированных чисел

Цель тестового понять что умеешь программировать: реализация + тесты. Сильно накручивать реализацию не нужно. Вас оценивают за скорость, и насколько быстро можете учиться. Мне не важно что вы умели до, мне важно как быстро вы учитесь и насколько сильно воркаете :)

Пример:
1. Отправили число 3
=> Записали в бд
=> Вернули в ответе массив [3]
2. Отправили число 2
=> Записали в БД
=> Вернули в ответ массив [2, 3]
3. Отправили число 1
=> Записали в БД
=> Вернули в ответ массив [1,2,3]

Возвращать ответ можно как угодно (жсоном, массивом), главное чтобы сортировалось и возвращалось
